#### Ruby のソースコードの取得方法

Ruby のソースコードの取得方法は主に以下の 2 つある。

1. [tarball](https://cache.ruby-lang.org/pub/ruby/4.0/) をダウンロード
1. Git リポジトリ（[ruby/ruby](https://github.com/ruby/ruby)） を clone（👈 今回はこちらの方法で取得）

#### Ruby のソースコードの取得方法の違い

前者の場合は、`configure` や `parse.c` が同梱されているため autotools と baseruby が不要。

後者の場合は、`configure` や `parse.c` の生成のために autotools や baseruby が必要。

[The Evolution of the CRuby Build System \- Speaker Deck](https://speakerdeck.com/kateinoigakukun/the-evolution-of-the-cruby-build-system?slide=9)

#### `./autogen.sh` の実行

このスクリプトを実行すると、`autoreconf` が実行されて `configure` スクリプトが生成される。

具体的には、`autoconf` が `configure.ac` というスクリプトを実行して、その結果 `configure` スクリプトが生成されるっぽい。

#### `../ruby/configure` の実行

`configure` スクリプトを実行すると `Makefile` が生成される。

```sh
$ ../ruby/configure --prefix=$PWD/../install --enable-shared
```

#### `../ruby/configure` のオプション

`--prefix` オプションはインストール先のディレクトリを指定する（このディレクトリに `ruby` コマンドや `libruby` 等が配置される）

`--enable-shared` オプションは共有ライブラリ `libruby.X.X.dylib` や `libruby.X.X.so` を動的リンクするためのオプション？

差分は以下の通り。

```
	/Users/keigo/workspace/shikumirb/workdir/build/../install/lib/libruby.4.1.dylib (compatibility version 4.1.0, current version 4.1.0)
```

`miniruby` と `ruby` は、`libruby` が動的リンクされているかどうかが異なる。

- `miniruby` は `libruby` が動的リンクされていない
- `ruby` は `libruby` が動的リンクされている

#### `make install` の実行前にビルド直後の `ruby` を実行

`make install` を実行する前に `build` ディレクトリの `ruby` を実行するとエラーになる。

```sh
~/workspace/rhc/workdir/build main
❯ ./miniruby -v
ruby 4.1.0dev (2026-03-27T07:09:03Z master 998e2f41c7) +PRISM [x86_64-darwin22]

~/workspace/rhc/workdir/build main
❯ ./ruby -v    
dyld[49288]: Library not loaded: /Users/keigo/workspace/rhc/workdir/build/../install/lib/libruby.4.1.dylib
  Referenced from: <95D8579B-24C6-35BC-8621-8610F3607DDF> /Users/keigo/workspace/rhc/workdir/build/ruby
  Reason: tried: '/Users/keigo/workspace/rhc/workdir/build/../install/lib/libruby.4.1.dylib' (no such file), '/System/Volumes/Preboot/Cryptexes/OS/Users/keigo/workspace/rhc/workdir/build/../install/lib/libruby.4.1.dylib' (no such file), '/Users/keigo/workspace/rhc/workdir/build/../install/lib/libruby.4.1.dylib' (no such file)
```

このエラーは以下のような流れで発生していそう。

- 動的リンクされている `install` ディレクトリの `libruby` を探しに行く
- `make install`が未実行だと `install` ディレクトリが存在しないためエラーになる
