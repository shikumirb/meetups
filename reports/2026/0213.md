## やったこと

今日は ruby コマンドの --dump オプションを利用して CRuby のデバッグ情報を出力してワイワイしました！

### `--dump` オプション

以下では Ruby のバージョン 3.4.5 を利用している。

```sh
❯ ruby -v
ruby 3.4.5 (2025-07-16 revision 20cda200d3) +PRISM [x86_64-darwin22]
```

`--dump` オプションを使うと、パーサが生成する AST、コンパイラが生成する ISeq を出力することができる。

```sh
❯ ruby --help | grep -A 6 "Dump List"
Dump List:
  insns           Instruction sequences.
  yydebug         yydebug of yacc parser generator.
  parsetree       Abstract syntax tree (AST).
  -optimize       Disable optimization (affects insns).
  +error-tolerant Error-tolerant parsing (affects yydebug, parsetree).
  +comment        Add comments to AST (affects parsetree with --parser=parse.y).
```

### ISeq の出力

`1` というスクリプトの ISeq は以下のようにして出力できる。

```sh
❯ ruby --parser=parse.y --dump=insns -e $'1'      
== disasm: #<ISeq:<main>@-e:1 (1,0)-(1,1)>
0000 putobject_INT2FIX_1_                                             (   1)[Li]
0001 leave
```

---

`'hello'` というスクリプトの ISeq は以下のようにして出力できる。

```sh
❯ ruby --parser=parse.y --dump=insns -e $'"hello"'                               
== disasm: #<ISeq:<main>@-e:1 (1,0)-(1,7)>
0000 putchilledstring                       "hello"                   (   1)[Li]
0002 leave
```

### AST と Iseq を予想する

`1 + 2 * 3` の AST は以下のような形になると予想される。（左側の子がレシーバ、右側の子が引数とする）

```
  +
 / \
1   *
   / \
  2   3
```

この AST を深さ優先で探索して帰りがけ順で命令を生成すると、以下のような命令列が生成されると予想される。

```
putobject 1
putobject 2
putobject 3
opt_mult
opt_plus
```

これを図示するとこんな感じ。`@ NODE_LIST` で引数がラップされているように見えるが、大体予想通り。

### ブロックとローカル変数

以下のような Ruby スクリプトを ISeq に変換してみる。

```ruby
10.times { |n| puts n }
```

ローカル変数に関する情報はローカルテーブルにスコープ単位で保存されるらしい。

上のスクリプトから生成される ISeq とローカルテーブルは大体次のようになりそう。

```
# トップレベルのスコープ
-------------------------------------------------
ISeq                                | Local table
-------------------------------------------------
putobject 10                        |
send      mid=:times, argc=0, block |
leave                               |
-------------------------------------------------

# ブロック内のスコープ
-------------------------------------------------
ISeq                                | Local table
-------------------------------------------------
putself                             | [1] n
getlocal  1                         |
send      mid=:puts,  argc=1        |
leave                               |
-------------------------------------------------
```

Ruby の処理系が生成する ISeq とローカルテーブルを確認する。

```sh
❯ ruby --parser=parse.y --dump=insns --dump=-optimize -e $'10.times { |n| puts n }'
== disasm: #<ISeq:<main>@-e:1 (1,0)-(1,23)>
0000 putobject                              10                        (   1)[Li]
0002 send                                   <calldata!mid:times, argc:0>, block in <main>
0005 leave

== disasm: #<ISeq:block in <main>@-e:1 (1,9)-(1,23)>
local table (size: 1, argc: 1 [opts: 0, rest: -1, post: 0, block: -1, kw: -1@-1, kwrest: -1])
[ 1] n@0<AmbiguousArg>
0000 putself                                                          (   1)[LiBc]
0001 getlocal                               n@0, 0
0004 send                                   <calldata!mid:puts, argc:1, FCALL|ARGS_SIMPLE>, nil
0007 leave                                  [Br]
```

これも大体予想通り。

### Tips

#### 最適化

`putobject_INT2FIX_1_` は最適化された命令。

`--dump=-optimize` を指定すると、以下のように `putobject_INT2FIX_1_` ではなく `putobject 1` が生成される。

```sh
❯ ruby --parser=parse.y --dump=insns --dump=-optimize -e $'1'
== disasm: #<ISeq:<main>@-e:1 (1,0)-(1,1)>
0000 putobject                              1                         (   1)[Li]
0002 leave
```

#### chilled string

`putchilledstring` は `chilled string` をスタックに積む命令。

`# frozen_string_literal` マジックコメントを追加すると、以下のように `putchilledstring` ではなく `putobject` や `putstring` が生成される。

```sh
❯ ruby --parser=parse.y --dump=insns -e $'# frozen_string_literal: true\n"hello"'
== disasm: #<ISeq:<main>@-e:1 (1,0)-(2,7)>
0000 putobject                              "hello"                   (   2)[Li]
0002 leave
```

```sh
❯ ruby --parser=parse.y --dump=insns -e $'# frozen_string_literal: false\n"hello"'
== disasm: #<ISeq:<main>@-e:1 (1,0)-(2,7)>
0000 putstring                              "hello"                   (   2)[Li]
0002 leave
```
