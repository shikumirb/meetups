### `--dump` オプション

`--dump=insns` オプションを使うと ISeq を出力できる。

例えば、以下のような Ruby スクリプトの ISeq を出力してみる。

```ruby
10.time do |n|
  puts n
end
```

```sh
❯ ruby --parser=parse.y --dump=insns -e $'10.times do \n |n| puts n \n end'
== disasm: #<ISeq:<main>@-e:1 (1,0)-(3,4)>
0000 putobject                              10                        (   1)[Li]
0002 send                                   <calldata!mid:times, argc:0>, block in <main>
0005 leave

== disasm: #<ISeq:block in <main>@-e:1 (1,9)-(3,4)>
local table (size: 1, argc: 1 [opts: 0, rest: -1, post: 0, block: -1, kw: -1@-1, kwrest: -1])
[ 1] n@0<AmbiguousArg>
0000 putself                                                          (   2)[LiBc]
0001 getlocal_WC_0                          n@0
0003 opt_send_without_block                 <calldata!mid:puts, argc:1, FCALL|ARGS_SIMPLE>
0005 leave                                                            (   3)[Br]
```

上の出力を見ると、以下のようになっていることがわかる。

- スコープごとに ISeq が分かれて出力される
  - `== disasm: #<ISeq:` から始まるブロックが 2 つ出力されている
- ブロックパラメータの情報がローカルテーブルに出力されている
  - `local table` から `[ 1] n@0<AmbiguousArg>` のあたり

### Ruby のコールスタック

`caller` を使うとバックトレースを見ることができる。

[Kernel\.\#caller \(Ruby 4\.0 リファレンスマニュアル\)](https://docs.ruby-lang.org/ja/latest/method/Kernel/m/caller.html)

`ruby` コマンドを引数なしで実行すると入力待ちの状態になるので、以下のようなスクリプトを入力してみる。

```ruby
❯ ruby
def foo = bar
def bar = baz
def baz = puts caller
foo
```

この状態で `Ctrl` + `D` を入力するとスクリプトが実行されて、コールスタックが出力される。

```ruby
❯ ruby
def foo = bar
def bar = baz
def baz = puts caller
foo
-:2:in 'Object#bar'
-:1:in 'Object#foo'
-:4:in '<main>'
```
