## やったこと

### `--dump` オプション

以下では Ruby のバージョン 3.4.5 を利用している。

```sh
❯ ruby -v
ruby 3.4.5 (2025-07-16 revision 20cda200d3) +PRISM [x86_64-darwin22]
```

`--dump` オプションを使うと、パーサが生成する AST、コンパイラが生成する ISeq を出力することができる。

```sh
❯ ruby --help | grep -A 6 "Dump List"
Dump List:
  insns           Instruction sequences.
  yydebug         yydebug of yacc parser generator.
  parsetree       Abstract syntax tree (AST).
  -optimize       Disable optimization (affects insns).
  +error-tolerant Error-tolerant parsing (affects yydebug, parsetree).
  +comment        Add comments to AST (affects parsetree with --parser=parse.y).
```

### ISeq の出力

`--dump=insns` というオプションを指定すると ISeq が出力される。

```sh
❯ ruby --parser=parse.y --dump=insns -e $'1 + 2 * 3'
== disasm: #<ISeq:<main>@-e:1 (1,0)-(1,9)>
0000 putobject_INT2FIX_1_                                             (   1)[Li]
0001 putobject                              2
0003 putobject                              3
0005 opt_mult                               <calldata!mid:*, argc:1, ARGS_SIMPLE>[CcCr]
0007 opt_plus                               <calldata!mid:+, argc:1, ARGS_SIMPLE>[CcCr]
0009 leave
```

左側の 4 桁の数字は、各命令が先頭から何バイトの位置にあるか（オフセット）を表しているっぽい。

このオフセットから各命令のバイト数がわかる。

| オフセット | 命令 | オペランド | バイト数 | 備考 |
| :--- | :--- | :--- | :--- | :--- |
| `0000` | `putobject_INT2FIX_1_` | なし | `1` | オペランドと命令がくっついている？<br>`putobject 1` だと 2 バイト必要なことに注意！ |
| `0001` | `putobject` | `2` | `2` | 命令とオペランドで 1 バイトずつ？ |
| `0003` | `putobject` | `3` | `2` | 命令とオペランドで 1 バイトずつ？ |
| `0005` | `opt_mult` | `<calldata!mid:*, argc:1, ARGS_SIMPLE>[CcCr]` | `2` | 命令とオペランドで 1 バイトずつ？ |
| `0007` | `opt_plus` | `<calldata!mid:+, argc:1, ARGS_SIMPLE>[CcCr]` | `2` | 命令とオペランドで 1 バイトずつ？ |
| `0009` | `leave` | なし | `?` | （上の結果からは読み取れないが）サイズは 1 バイト？ |

### ISeq の出力（最適化なし）

`--dump=insns --dump=-optimize` というオプションを指定すると、最適化されていない ISeq が出力される。

```sh
❯ ruby --parser=parse.y --dump=insns --dump=-optimize -e $'1 + 2 * 3'
== disasm: #<ISeq:<main>@-e:1 (1,0)-(1,9)>
0000 putobject                              1                         (   1)[Li]
0002 putobject                              2
0004 putobject                              3
0006 send                                   <calldata!mid:*, argc:1, ARGS_SIMPLE>, nil
0009 send                                   <calldata!mid:+, argc:1, ARGS_SIMPLE>, nil
0012 leave
```

このオフセットから各命令のバイト数がわかる。

最適化を外すと、一部のバイトコードのサイズが大きくなっていることに注意！

| オフセット | 命令 | オペランド | バイト数 | 備考 |
| :--- | :--- | :--- | :--- | :--- |
| `0000` | `putobject` | `1` | `2` | 命令とオペランドで 1 バイトずつ？ |
| `0002` | `putobject` | `2` | `2` | 命令とオペランドで 1 バイトずつ？ |
| `0004` | `putobject` | `3` | `2` | 命令とオペランドで 1 バイトずつ？ |
| `0006` | `send` | `<calldata!mid:*, argc:1, ARGS_SIMPLE>, nil` | `3` | 命令と 2 つのオペランドそれぞれ 1 バイトずつ？ |
| `0009` | `send` | `<calldata!mid:+, argc:1, ARGS_SIMPLE>, nil` | `3` | 命令と 2 つのオペランドそれぞれ 1 バイトずつ？ |
| `00012` | `leave` | なし | `?` | （上の結果からは読み取れないが）サイズは 1 バイト？ |
