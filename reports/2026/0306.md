## やったこと

今日も ruby コマンドの --dump オプションを利用して CRuby のデバッグ情報を出力してワイワイしました！

### `--dump` オプション

以下では Ruby のバージョン 3.4.5 を利用している。

```sh
❯ ruby -v
ruby 3.4.5 (2025-07-16 revision 20cda200d3) +PRISM [x86_64-darwin22]
```

`--dump` オプションを使うと、パーサが生成する AST、コンパイラが生成する ISeq を出力することができる。

```sh
❯ ruby --help | grep -A 6 "Dump List"
Dump List:
  insns           Instruction sequences.
  yydebug         yydebug of yacc parser generator.
  parsetree       Abstract syntax tree (AST).
  -optimize       Disable optimization (affects insns).
  +error-tolerant Error-tolerant parsing (affects yydebug, parsetree).
  +comment        Add comments to AST (affects parsetree with --parser=parse.y).
```

### メソッド定義のローカルテーブル

メソッド定義のローカルテーブルを出力してみる。

```sh
❯ ruby --parser=parse.y --dump=insns --dump=-optimize -e $'def add(a, b) \n sum = a + b \n end'
== disasm: #<ISeq:<main>@-e:1 (1,0)-(3,4)>
0000 definemethod                           :add, add                 (   1)[Li]
0003 putobject                              :add
0005 leave

== disasm: #<ISeq:add@-e:1 (1,0)-(3,4)>
local table (size: 3, argc: 2 [opts: 0, rest: -1, post: 0, block: -1, kw: -1@-1, kwrest: -1])
[ 3] a@0<Arg>   [ 2] b@1<Arg>   [ 1] sum@2
0000 getlocal                               a@0, 0                    (   2)[LiCa]
0003 getlocal                               b@1, 0
0006 send                                   <calldata!mid:+, argc:1, ARGS_SIMPLE>, nil
0009 dup
0010 setlocal                               sum@2, 0
0013 leave                                                            (   3)[Re]
```

ローカルテーブルの部分だけを取り出すと以下のようになる。

```sh
local table (size: 3, argc: 2 [opts: 0, rest: -1, post: 0, block: -1, kw: -1@-1, kwrest: -1])
[ 3] a@0<Arg>   [ 2] b@1<Arg>   [ 1] sum@2
```

index と @ の数字を足すと 3 になり、スコープ内の変数の数と一致するように見える。

| index | 変数名 | @ | ラベル |
| :--- | :--- | :--- | :--- |
| 1 | `sum` | 2 | なし |
| 2 | `b` | 1 | \<Arg\> |
| 3 | `a` | 0 | \<Arg\> |

### メソッド定義のローカルテーブル（可変長引数 / rest 引数）

`*` のついた引数は可変長引数や rest 引数と呼ばれる。

```sh
# a=1, b=2, c=[3, 4], d=5
❯ ruby -e $'def foo(a, b, *c, d) \n a + b + c.size + d \n end \n p foo(1, 2, 3, 4, 5)'
10

# a=1, b=2, c=[3, 4, 5], d=6
❯ ruby -e $'def foo(a, b, *c, d) \n a + b + c.size + d \n end \n p foo(1, 2, 3, 4, 5, 6)'
12

# a=1, b=2, c=[3, 4, 5, 6], d=7
❯ ruby -e $'def foo(a, b, *c, d) \n a + b + c.size + d \n end \n p foo(1, 2, 3, 4, 5, 6, 7)'
14
```

このメソッド定義のローカルテーブルを出力してみる。

```sh

0005 leave

[ 4] a@0<Arg>   [ 3] b@1<Arg>   [ 2] c@2<Rest>  [ 1] d@3<Post>
```

index と @ の数字を足すと 4 になり、スコープ内の変数の数と一致するように見える。

| index | 変数名 | @ | ラベル |

| 1 | `d` | 3 | \<Post\> |
| 2 | `c` | 2 | \<Rest\> |
| 3 | `b` | 1 | \<Arg\> |
| 4 | `a` | 0 | \<Arg\> |
