### バージョン情報の出力メッセージを追加

Ruby のバージョン出力にメッセージを追加する。

`version.c`というファイルの `ruby_show_version` 関数を修正すれば良さそう。

```c
void
ruby_show_version(void)
{    
    puts(rb_dynamic_description);

#ifdef RUBY_LAST_COMMIT_TITLE
    fputs("last_commit=" RUBY_LAST_COMMIT_TITLE, stdout);
#endif
#ifdef HAVE_MALLOC_CONF
    if (malloc_conf) printf("malloc_conf=%s\n", malloc_conf);
#endif
    fflush(stdout);
}       
```

具体的には、`fflush(stdout);` の直前に `printf("適当なメッセージ\n");` のような処理を追加してみる。

### `Array` クラスの C 言語のソースコード

Ruby の `Array` クラスは `array.c` で実装されているっぽい。

ざっと見た感じ、`Init_Array` 関数の中で Ruby のメソッドと C の関数を紐づけているように見える。

```c
Init_Array(void)
{
(中略)
    rb_define_method(rb_cArray, "initialize", rb_ary_initialize, -1);
    rb_define_method(rb_cArray, "initialize_copy", rb_ary_replace, 1); 
 
    rb_define_method(rb_cArray, "inspect", rb_ary_inspect, 0);
    rb_define_alias(rb_cArray,  "to_s", "inspect");
    rb_define_method(rb_cArray, "to_a", rb_ary_to_a, 0);
    rb_define_method(rb_cArray, "to_h", rb_ary_to_h, 0);
    rb_define_method(rb_cArray, "to_ary", rb_ary_to_ary_m, 0);
````

具体的には `rb_define_method` の引数に以下のように値を渡しているように見える。

- 第 1 引数：Ruby のクラス名
- 第 2 引数：Ruby のメソッド名
- 第 3 引数：C の関数名（👈 この関数が具体的な処理の中身）
- 第 4 引数：Ruby のメソッドの引数の数

### `Array` クラスの `first` メソッドに対応する C 言語の関数を探す

`array.c` の中で `Array` クラスの `first` メソッドに対応する関数を探してみる。

ざっと見た感じ、以下の 2 つの関数が怪しそう。

```c
#if 0
static VALUE
rb_ary_first(int argc, VALUE *argv, VALUE ary)
{   
    if (argc == 0) {
        if (RARRAY_LEN(ary) == 0) return Qnil;
        return RARRAY_AREF(ary, 0);
    }
    else {
        return ary_take_first_or_last(argc, argv, ary, ARY_TAKE_FIRST);
    }
}
#endif 
        
static VALUE
ary_first(VALUE self)
{   
    return (RARRAY_LEN(self) == 0) ? Qnil : RARRAY_AREF(self, 0);
}
```

前者の `rb_ary_first` 関数は `#if 0`と `#endif` ディレクティブで囲まれているため、プリプロセッサによって無視されてコンパイル対象から外れる。
（[suketam](https://github.com/suketa) さんから教えていただきました、ありがとうございます！）

後者の `ary_first` 関数は `Init_Array` 関数の中 `rb_define_method` の引数に渡されていない。

`Array` クラスの `first` メソッドと `ary_first` 関数の間に何か関係があるのかパッとわからない。。

### Array クラスの first メソッドに対応する C 言語の関数がどうやって実行されているか

[\@shirakabesoft](https://x.com/shirakabesoft) さんから「`array.rb` から C 言語の `ary_first` 関数を呼んでいるのではないか」という情報を共有いただいたので確認してみる。

確かに `array.rb` の以下で、`ary_first` 関数と `ary_take_first_or_last_n` 関数を呼んでそう。

```ruby
  def first n = unspecified = true
    if Primitive.mandatory_only?
      Primitive.attr! :leaf
      Primitive.cexpr! %q{ ary_first(self) }
    else
      if unspecified 
        Primitive.cexpr! %q{ ary_first(self) }
      else
        Primitive.cexpr! %q{  ary_take_first_or_last_n(self, NUM2LONG(n), ARY_TAKE_FIRST) }
      end
    end
  end
```
